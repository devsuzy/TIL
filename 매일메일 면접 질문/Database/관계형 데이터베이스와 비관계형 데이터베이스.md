#  관계형 데이터베이스와 비관계형 데이터베이스

> 매일메일 FE 2026년 1월 4주차 화요일 질문

### 관계형 데이터베이스
- 데이터를 테이블 형식으로 저장하고 관리하는 데이터베이스
- 각 테이블은 고정된 고유의 스키마를 가짐
- 데이터 타입과 구조가 엄격하게 정의되어 있음
- MySQL, PostgresSQL, Oracle

#### 장점
- 데이터 간의 관계를 명확하게 표현 가능
- 정형화된 데이터를 다룰 때 유용함
- 복잡한 조건의 데이터 조회나 조인 처리가 용이함
#### 단점
- 스키마가 고정되어있기 때문에 유연성이 떨어짐
- 테이블 구조를 바꾸려면 과정이 복잡하고 시간이 오래걸림
- 서버를 여러 대로 분산시키는 수평적 확장이 상대적으로 어려움
- 대규모 트래픽을 처리해야하는 시스템에 확장성 한계로 작용할 수 있음

### 비관계형 데이터베이스
- 전통적인 테이블 기반 구조가 아닌 유연한 데이터 모델에 사용
- 스키마가 고정되어있지 않아 저장되는 데이터 구조가 일관되지 않음
- MongoDB, Cassandra, Redis

#### 장점
- 대용량 데이터를 빠르게 처리함
- 데이터 구조가 자주 변경될 가능성이 높은 프로젝트에 적합함
- 관계를 맺고 조인하는 대신 한 번의 조회로 필요한 정보를 가져올 수 있음
#### 단점
- 같은 컬렉션 안에 구조가 다른 데이터들이 들어갈 수 있어 나중에 검증할 때 오류 발생 가능성이 높음
- 조인 기능이 제한적이기 때문에 데이터 간 관계 표현에 한계가 있음
