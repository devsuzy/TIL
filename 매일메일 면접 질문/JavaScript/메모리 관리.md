# 자바스크립트의 메모리 관리

> 매일메일 FE 2026년 1월 3주차 화요일 질문

### 메모리 관리
- C와 같은 저수준 언어와 달리 자동으로 할당 및 해제가 됨
- 변수나 객체가 생성할 때 자동으로 메모리를 할당
- 할당된 메모리가 더 이상 필요하지 않을 때 메모리 자동 해제

### 메모리 할당
- 문자열, 숫자와 같은 원시 값은 고정 크기를 가지고 있기 때문에 정적 데이터라고 불리며, 이는 Stack 영역에 저장
```js
const n = 123; // 정수를 담기 위한 메모리 할당
const s = "azerty"; // 문자열을 담기 위한 메모리 할당
```
- 객체, 배열과 같은 참조 타입은 실행 시점에 필요한 만큼 메모리가 동적 할당되기 때문에 동적 데이터라고 불리며, 이는 Heap 영역에 저장
```js
const o = {
  a: 1,
  b: null,
}; // 오브젝트와 그 오브젝트에 포함된 값들을 담기 위한 메모리 할당

const a = [1, null, "abra"]; // 배열과 배열에 담긴 값들을 위한 메모리 할당

```

### 가비지 콜렉션(GC)
- 언어 차원에서 메모리 할당을 추적하고, 특정 메모리가 필요하지 않게 되었다면 메모리를 해제하는 방식

### Reference-counting 가비지 콜렉션
- 가장 단순하게 구현된 알고리즘
- 객체가 참조되는 횟수를 추적하고 참조 횟수가 0이 되면 메모리에서 해제
- 순환 참조에서는 참조 카운트가 0이 되지 않아 메모리에서 해제되지 않음 -> 메모리 누수 발생

### Mark-and-sweep 가비지 콜렉션
- 도달할 수 없는 객체를 메모리 해제하는 알고리즘
- 루트 객체에서 시작하여 참조되는 객체를 표시(mark)하고, 마지막까지 표시되지 않은 객체를 찾아 메모리를 청소(sweep)하는 구조
- 순환 참조가 발생하더라도 루트에서 도달이 불가능하다면 표시가 되지 않음
